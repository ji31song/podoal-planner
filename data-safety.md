# 포도알플래너 Google Play Data Safety 작성 초안

작성일: 2026-07-18
운영자: 지송
문의: yoojin24song@gmail.com

> 이 문서는 Play Console 입력을 돕기 위한 내부 체크리스트다. 실제 제출 직전에 배포된 Android 앱, Firebase 설정, 추가 SDK 및 개인정보처리방침과 다시 대조해야 한다.

## 1. 앱의 데이터 처리 개요

- 가족을 처음 만드는 소유자는 Firebase Authentication의 Google 로그인을 사용한다. 초대 코드로 참여하는 구성원은 Google 로그인 없이 자동 생성되는 Firebase 익명 참여 계정을 사용한다.
- 가족 이름, 구성원, 일정, 체크리스트, 메모, 댓글, 포도알 기록과 이용자가 직접 입력한 전화번호를 Firebase Realtime Database에 저장한다.
- Firebase Cloud Functions가 가족 가입, 탈퇴, 소유권 이전과 계정 삭제를 처리한다.
- Firebase App Check와 Google Play Integrity가 비정상 앱·요청을 판별하기 위한 앱·기기 무결성 신호를 처리한다.
- Firebase 서비스 운영 과정에서 IP 주소, 사용자 에이전트, 운영체제·기기 모델·앱/SDK 버전 등의 기술 정보가 자동 처리될 수 있다.
- 같은 가족으로 승인된 구성원은 가족 데이터를 서로 열람하며 역할에 따라 수정할 수 있다.
- 광고, 맞춤형 광고, 마케팅, 위치 추적 또는 연락처 자동 수집 기능은 현재 없다.
- Google Analytics, Crashlytics와 별도 광고 SDK는 현재 프로젝트에서 확인되지 않는다.

## 2. Play Console의 주요 질문 초안

| 질문 | 권장 답변 | 근거 및 확인 사항 |
|---|---|---|
| 앱이 사용자 데이터를 수집하거나 공유하는가? | 예 | 로그인 및 가족 데이터를 기기 밖 Firebase로 전송한다. |
| 수집 데이터가 전송 중 암호화되는가? | 예 | Firebase SDK의 HTTPS/TLS 통신을 사용하며 Android 앱은 평문 통신을 차단한다. |
| 사용자가 데이터 삭제를 요청할 수 있는가? | 예 | 앱 내부 계정 삭제와 `delete-account.html` 외부 요청 페이지가 있다. |
| 모든 사용자가 계정을 만들어야 하는가? | 예 | 소유자는 Google 계정으로, 초대 구성원은 앱이 자동 생성하는 익명 Firebase 계정으로 식별된다. 초대 구성원에게 별도의 로그인 절차나 이메일은 요구하지 않는다. |
| 데이터가 제3자에게 판매되는가? | 아니요 | 판매 기능이 없다. |
| 광고 목적으로 데이터를 사용하는가? | 아니요 | 광고 SDK와 광고 기능이 없다. |
| 독립적인 보안 검토를 받았는가? | 아니요 | 외부 공인 보안 심사를 받았다는 증빙이 없다. |

## 3. 데이터 유형별 신고 초안

| Play 데이터 범주 | 구체적인 데이터 | 수집 | 공유 | 필수 여부 | 사용 목적 |
|---|---|---:|---:|---|---|
| 개인정보 · 이름 | 소유자의 Google 계정 표시 이름, 가족 구성원 이름 | 예 | 조건부 | Google 이름은 소유자만 해당, 구성원 이름은 사용자 입력 | 앱 기능, 계정 관리 |
| 개인정보 · 이메일 주소 | 소유자의 Google 로그인 이메일 | 예 | 아니요 | 가족 소유자에게만 필수, 초대 구성원에게는 수집하지 않음 | 인증, 계정 관리, 보안 |
| 개인정보 · 사용자 ID | Firebase UID, 가족·구성원 연결 ID | 예 | 아니요 | 필수 | 인증, 계정 관리, 보안, 가족 분리 |
| 개인정보 · 전화번호 | 이용자가 전화번호부에 직접 입력한 번호 | 예 | 조건부 | 선택 | 앱 기능, 가족 내 공유 |
| 앱 활동 · 기타 사용자 생성 콘텐츠 | 일정, 체크리스트, 메모, 댓글, 포도알 기록 | 예 | 조건부 | 선택 | 앱 기능, 가족 내 공유 |
| 기기 또는 기타 식별자·보안 신호 | IP 주소, Firebase 앱 식별 정보, 사용자 에이전트, OS·기기·앱/SDK 버전, Play Integrity 무결성 토큰 | 예 | 아니요 | 자동 | 인증, 보안, 사기·악용 방지 |

> 위 기술 정보가 Play Console의 어느 세부 범주에 해당하는지는 제출 화면의 최신 정의와 대조한다. 현재 앱에는 별도 Analytics·Crashlytics SDK가 없으므로 사용 행동 분석이나 충돌 보고 목적의 데이터 수집으로 신고하지 않는다.

### ‘공유’ 항목 최종 판단 시 주의

- Firebase는 앱을 대신해 데이터를 처리하는 서비스 제공업체이므로 Google Play 정의상 일반적인 제3자 공유에서 제외될 수 있다.
- 가족 구성원에게 보여주는 데이터는 사용자가 기대하고 실행한 앱 내부 공유다. Play Console의 최신 설명에서 ‘사용자에 의해 시작된 전송’ 예외 적용 여부를 최종 제출 시 확인한다.
- 예외 적용이 확실하지 않다면 이름, 전화번호 및 사용자 생성 콘텐츠를 ‘공유됨’으로 보수적으로 신고한다.

## 4. 데이터별 세부 답변

### 이름, 이메일 주소, 사용자 ID

- 수집 여부: 예
- 일시적 처리 여부: 아니요
- 필수 여부: Firebase 사용자 ID는 모든 이용자에게 필수. Google 로그인 정보와 이메일은 가족을 만드는 소유자에게만 필수
- 목적: 앱 기능, 계정 관리, 보안 및 가족별 접근 제어
- 판매: 아니요

### 전화번호

- 수집 여부: 예
- 자동 연락처 접근: 아니요
- 입력 방식: 사용자가 앱 전화번호부에 직접 입력
- 필수 여부: 선택
- 목적: 가족 전화번호부 기능
- 가족 구성원 열람: 예
- 판매 및 광고 이용: 아니요

### 일정·체크리스트·메모·댓글·포도알 기록

- 수집 여부: 예
- 필수 여부: 선택
- 목적: 플래너와 가족 공유 기능
- 같은 가족 구성원 열람: 예
- 판매 및 광고 이용: 아니요

### 앱·기기 기술 정보와 무결성 신호

- 수집 여부: 예, Firebase 서비스와 App Check가 자동 처리
- 구체적 항목: IP 주소, 사용자 에이전트, 운영체제·기기 모델·앱/SDK 버전, Firebase 앱 식별 정보, Play Integrity 무결성 토큰
- 목적: 로그인 인증, 가족 데이터 보호, 비정상 앱과 자동화된 공격 방지
- 광고·마케팅 이용: 아니요
- 판매: 아니요

## 5. 보관 및 삭제 설명

- 계정과 가족 데이터는 서비스 이용 중 Firebase에 보관한다.
- 초대 구성원이 기기 연결을 해제하면 가족 접근 권한과 익명 참여 계정을 제거하고 가족 공동 데이터는 유지한다.
- 가족 소유자가 분실·교체된 기기의 연결을 끊으면 해당 기기의 가족 접근 권한과 계정–구성원 연결을 제거하고, 구성원 프로필과 가족 공동 데이터는 유지한다.
- 계정 삭제 시 Firebase 인증 계정, 가족 연결 및 해당 계정에 연결된 구성원 프로필·일정·체크리스트·메모·전화번호부 데이터를 삭제한다.
- 소유자만 남은 가족을 삭제하면 가족 데이터와 초대 코드를 삭제한다.
- 다른 가족 구성원이 작성한 공동 데이터는 해당 가족에 남을 수 있다.
- Firebase Authentication의 일부 운영 기록과 백업 사본은 Google의 보관 정책에 따라 제한된 기간 동안 남은 뒤 삭제될 수 있다.
- 앱에 로그인할 수 없는 사용자는 공개 삭제 요청 페이지에서 이메일로 삭제를 요청할 수 있다.

## 6. Play Console에 사용할 공개 URL

배포 후 실제 HTTPS 주소로 교체한다.

- 개인정보처리방침: `https://podoal-planner-c1d1e.web.app/privacy.html`
- 이용약관: `https://podoal-planner-c1d1e.web.app/terms.html`
- 계정 삭제 요청: `https://podoal-planner-c1d1e.web.app/delete-account.html`
- 지원 및 문의: `https://podoal-planner-c1d1e.web.app/support.html`

로컬 파일 경로나 `file://` 주소는 Play Console에 제출할 수 없다.

- [x] Play Console 개발자 이름과 개인정보처리방침의 운영자 이름이 일치한다.

- [x] Firebase Authentication, Realtime Database 및 Cloud Functions의 운영 프로젝트를 확인했다.
- [x] Analytics, Crashlytics, 광고 또는 다른 SDK가 추가되었는지 검사했다.
- [x] Android 권한 목록에 연락처, 위치, 저장소 등 불필요한 권한이 없는지 확인했다.
- [x] Data Safety 답변과 개인정보처리방침의 수집·공유·삭제 설명을 일치시켰다.
